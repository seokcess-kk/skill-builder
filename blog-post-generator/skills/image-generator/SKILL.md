# 범용 블로그 이미지 생성기

네이버 블로그용 장문 콘텐츠를 설득 퍼널 구조로 재편집하고, 섹션별 디자인 PNG 이미지로 제작합니다.

**Input**: 원고 + 브랜드 정보 (홈페이지 URL 권장)
**Output**: 섹션별 PNG 이미지 (680px, 모바일 최적화)

---

## Step 0: 사용자 입력 수집

| 항목 | 설명 | 미제공 시 |
|------|------|-----------|
| ★ 원고 | 블로그 원고 전문 또는 주제/키워드 | 주제만 있으면 초안 생성 후 진행 |
| ★ 브랜드명 | 업체명 또는 서비스명 | 필수 |
| ★ 홈페이지 URL | 브랜드 웹사이트 | 필수 (색상/톤 추출) |
| 영문 브랜드명 | CTA/disclaimer용 | 홈페이지에서 추출 또는 음역 |
| 산업/업종 | 의료, 뷰티, 교육, 법률 등 | 홈페이지/원고에서 자동 감지 |
| 타겟 페르소나 | 주요 독자층 | 원고 문맥에서 추론 |
| 메인 키워드 | 네이버 SEO 타겟 | 브랜드+업종으로 조합 |
| 브랜드 컬러 | HEX 코드 | 홈페이지에서 추출 |
| 디자인 테마 | editorial/warm/clean/bold | 자동 감지 |
| 전문가 이름/직함 | 인터뷰이 | 없으면 브랜드 소개로 대체 |

## Step 0.5: 홈페이지 브랜드 분석 (URL 제공 시)

`WebFetch`로 홈페이지를 읽고 아래 항목을 추출합니다:

### 추출 항목
1. **브랜드 컬러**: 로고, 헤더, 버튼, 주요 UI에서 Primary/Accent 컬러 HEX 추출
2. **톤앤매너**: 카피 스타일(격식/친근/권위/공감), 전체 분위기
3. **USP/슬로건**: 메인 슬로건, 차별화 메시지, 핵심 가치
4. **산업/타겟 확인**: 업종, 서비스 대상, 고객층
5. **규제 업종 감지**: 의료/금융/법률 여부 → 해당 시 컴플라이언스 가이드 자동 로드

### 컬러 추출 → CSS 변수 매핑

```css
:root {
  --brand-primary: {홈페이지 메인 컬러};
  --brand-accent:  {홈페이지 포인트 컬러};
  --brand-mid:     {primary-accent 중간 블렌드};
  --bg-tint:       {primary 95% 밝기 틴트};
  --bg-dark:       {primary 다크 버전};
}
```

추출 결과를 사용자에게 요약 보고 후 확인받고 진행합니다.

## Step 1: 원고 분석 및 전환 최적화 폴리싱

### 1-1. 원고 분석

- **핵심 주제**: 무엇을 해결/제공하는가
- **타겟 감지**: 독자층의 고민/상황
- **톤 감지**: 공식적/친근/권위적/공감형
- **USP 추출**: 차별화 포인트 1~3가지
- **기존 구조 분석**: 원고가 어느 퍼널 단계를 커버하는지

### 1-2. 전환 최적화 폴리싱

원본 내용 유지하되 전환율 중심으로 재편집합니다:

- **Hook 강화**: 첫 문장이 스크롤을 멈추는가 → 아니면 재작성
- **Pain Point 선명화**: 문제를 구체적, 감정적으로 표현
- **Why 설득력**: 원인 설명에 숫자/비교/전문 용어(쉬운 설명 병행)
- **Solution 차별화**: USP 기반 단계형 또는 비교형으로 재구성
- **Proof 신뢰도**: 수치, 후기가 과장 없이 구체적인지
- **CTA 명확성**: 다음 행동이 1가지로 명확한지

### 1-3. 규제 업종 컴플라이언스 적용

의료 업종 감지 시 → `references/medical-ad-compliance.md` 로드하여 원고 전체를 검수합니다.
금융/법률 업종 감지 시 → 해당 업종 고지 기준을 적용합니다.

> 폴리싱 완료 후 "원고 분석 요약"을 사용자에게 보고합니다.
> (감지된 산업, 톤, 선택 테마, 컴플라이언스 적용 여부 포함)

## Step 2: 퍼널 구조 세분화

폴리싱된 원고를 10개 섹션으로 매핑합니다. 없는 내용은 새로 작성합니다.

| 섹션 | 역할 | 분량 |
|------|------|------|
| 01 Hook | 스크롤 정지 인용문/선언 | 1~2문장 |
| 02 Intro | 브랜드/전문가 신뢰 구축 | 프로필 + 3~5문장 |
| 03 Pain | 독자 공감 (페인포인트) | 300~500자 |
| 03b Bridge | Pain → Why 전환 | 1~2문장 |
| 04 Why | 원인 분석 (전문성 기반) | 400~600자 |
| 04b Bridge | Why → Solution 전환 | 1~2문장 |
| 05 Solution | 차별화 솔루션 제시 | 500~800자 |
| 06 Proof | 신뢰 근거 (수치/후기/사례) | 200~400자 |
| 07 CTA | 행동 유도 (1가지 명확 행동) | 브랜딩 문구 + 연락처 |
| 10 Disclaimer | 업종별 법적 고지 | 유형별 자동 |

**파일 네이밍**: `{brand-slug}-01-hook.html` ~ `{brand-slug}-10-disclaimer.html`

**폴더 구조**:
```
output/
├── {brand-slug}/
│   ├── html/    ← HTML 중간 파일 (디자인 적용된 소스)
│   └── png/     ← PNG 최종 결과물 (블로그 업로드용)
```

## Step 3: 디자인 테마 및 변형 자동 선택

### 3-1. 테마 자동 감지

| 감지 조건 | 테마 | 특징 |
|-----------|------|------|
| 의료/법률/금융 + 권위적 톤 | `editorial` | Serif 주도, 절제된 프리미엄 |
| 뷰티/라이프스타일 + 공감형 | `warm` | Serif+Sans, 따뜻한 파스텔 |
| 교육/커머스 + 정보형 | `clean` | Sans 주도, 미니멀 그리드 |
| 스타트업/IT + 임팩트형 | `bold` | 강한 타이포, 대비 강조 |

사용자 지정 시 해당 테마 우선. 홈페이지 분석 결과와 결합하여 최적 테마 결정.

### 3-3. 액센트 모드

같은 브랜드 컬러라도 **사용 방식을 회전**하여 매번 다른 분위기를 연출합니다.
HTML 생성 시 `<body class="accent-{모드명}">` 클래스를 적용합니다.

| 모드 | 클래스 | 컬러 사용 방식 | 분위기 |
|------|--------|---------------|--------|
| Primary 주도 | (기본, 클래스 없음) | primary=배경·강조, accent=포인트 | 브랜드 정통 |
| Accent 주도 | `accent-swap` | accent=배경·강조, primary=포인트 | 신선·활력 |
| Neutral 주도 | `accent-neutral` | 회색/백색 기반, 양쪽 컬러 모두 텍스트·포인트에만 | 톤다운·세련 |
| Gradient | `accent-gradient` | primary→accent 그라디언트를 다크 배경에 적용 | 역동·프리미엄 |

**CSS 변수 오버라이드** (예시 — 실제 값은 브랜드 컬러에서 파생):

```css
/* Accent 주도: primary↔accent 역할 교환 */
body.accent-swap {
  --brand-primary: {원래 accent};
  --brand-accent:  {원래 primary};
  --brand-mid:     {새 블렌드};
  --bg-tint:       {accent 기반 95% 밝기};
  --bg-dark:       {accent 다크 버전};
}

/* Neutral 주도: 배경을 무채색 계열로 */
body.accent-neutral {
  --bg-tint:  #F7F7F8;
  --bg-warm:  #F3F3F4;
  --bg-dark:  #2D2D2D;
  --border-soft: #E8E8E8;
}

/* Gradient: 다크 배경을 그라디언트로 */
body.accent-gradient {
  --bg-dark: linear-gradient(135deg, {primary} 0%, {accent} 100%);
}
```

### 3-4. 컴포지션 레시피

섹션별 개별 선택 대신, **포스트 전체의 변형 조합을 레시피로 관리**합니다.
같은 브랜드라도 다른 레시피를 적용하면 전혀 다른 시각적 경험이 됩니다.

#### 조합 설계 원칙

레시피는 아래 원칙에 따라 조합되어 있으며, 커스텀 조합 시에도 이 원칙을 따릅니다.

**① 밝기 리듬 (다크/라이트 교차)**
소비자가 스크롤할 때 시각적 단조로움을 피하는 핵심 장치입니다.
- 다크 배경 섹션은 전체 7개 중 최대 3개, 연속 2개까지만 허용
- Hook과 CTA가 같은 톤이면 양 끝이 호응하여 완결감이 생깁니다
- 중간부(Why/Solution)에서 다크↔라이트 전환이 있으면 "전환점"이 강조됩니다

**② 감정 곡선 매칭**
블로그 설득 퍼널은 자체 감정 곡선을 가집니다. 레이아웃이 이 곡선과 일치해야 자연스럽습니다.
```
관심(Hook) → 신뢰(Intro) → 공감(Pain) → 불안·이해(Why) → 안도(Solution) → 확신(Proof) → 행동(CTA)
```
- Hook: 시선을 멈추는 장치 → 임팩트형(B) 또는 여운형(E) 모두 가능
- Pain: 감정이 가장 깊어지는 구간 → 스토리(C)나 voice card(A)로 몰입 유도
- Why: 전환점("왜?") → 다크 배경이 적합 (시선 집중 + 긴장감)
- Solution: 안도 → 라이트로 전환하면 "해결" 느낌 강화
- CTA: 행동 유도 → 명확한 구조 (다크 권위형 or 라이트 친근형)

**③ 정렬 변화 (시선 흐름)**
같은 중앙 정렬이 계속되면 지루합니다.
- 중앙 → 좌우비대칭 → 중앙 → 그리드 식으로 정렬 변화를 줍니다
- Hook과 CTA가 같은 정렬 패턴이면 시각적 완결감이 생깁니다

#### 레시피 선택 규칙

- 같은 브랜드의 이전 포스트에서 사용한 레시피를 피합니다
- 원고의 톤/구조가 레시피의 감정 곡선과 맞는 것을 우선 선택합니다
- 사용자가 직접 지정할 수도 있습니다
- Bridge 변형은 레시피와 무관하게 앞뒤 섹션의 밝기에 따라 독립 선택합니다

#### 레시피 목록

**① 클래식 에디토리얼** — 점진적 권위 구축
- 액센트: Primary 주도
- 적합: 의료/법률/금융, 공식적 톤, 전문가 프로필이 풍성한 원고

| Hook | Intro | Pain | Why | Solution | Proof | CTA |
|------|-------|------|-----|----------|-------|-----|
| A 라이트 | A 숫자+자격 | B Q&A | A **다크** 그리드 | A 라이트 타임라인 | A 통계+리뷰 | A **다크** 코너장식 |

```
밝기: 라 → 라 → 라 → (Bridge-B 다크) → ■다크■ → (Bridge-A 라이트) → 라 → 라 → ■다크■
감정: 정제된 인용 → 권위 프로필 → Q&A 공감 → ──전환── → 데이터 원인 → ──전환── → 체계적 해결 → 수치 증거 → 격식 CTA
```
Why 다크로 "전환점" 강조, CTA 다크가 Hook-A 코너장식과 호응하여 완결감.

---

**② 웜 내러티브** — 감성 공감 스토리
- 액센트: Neutral 주도
- 적합: 뷰티/라이프스타일, 공감형 원고, 여성 타겟

| Hook | Intro | Pain | Why | Solution | Proof | CTA |
|------|-------|------|-----|----------|-------|-----|
| C 라이트 비대칭인용 | B 아이콘그리드 | C 스토리텔링 | B 라이트 아이콘카드 | B 라이트 어프로치 | B 코멘트형 | B 라이트 풀컬러 |


- 액센트: Accent Swap
- 적합: 스타트업/IT, 젊은 타겟, 임팩트 필요한 원고

## Step 4: HTML 파일 생성

### 공통 기술 규칙

- **너비**: 680px 고정
- **폰트**: `'Noto Serif CJK KR'` (헤드라인/인용) + `'Noto Sans CJK KR'` (본문)
- **외부 리소스 금지**: 모든 CSS는 인라인 또는 `<style>` 태그
- **이미지 placeholder**: CSS gradient + 프레임 (실제 사진 제공 시 `<img>` 교체)
- **다크/라이트 리듬**: 라이트 섹션 3개 이상 연속 금지
- **줄바꿈 규칙**:
  - `body { word-break: keep-all }` 필수
  - 680px 기준 1줄 약 20~25자, 의미 단위에서 `<br>` 수동 삽입
  - 조사(은/는/이/가/를/에)가 줄 첫머리로 넘어가지 않도록 구 단위 끊기
  - `<strong>` 강조 텍스트는 줄 걸침 없이 한 줄 완결

### CSS 클래스 체계

```html
<div class="hook hook-a theme-editorial">
<div class="intro intro-b theme-warm">
```

각 변형 클래스는 `assets/base-styles.css`의 공통 컴포넌트를 상속하며, 변형별 레이아웃만 오버라이드합니다.

### body 클래스 적용 (필수)

모든 섹션 HTML의 `<body>` 태그에 테마+액센트 클래스를 적용해야 합니다. `base-styles.css`의 테마별 CSS 변수(`--theme-*`)가 이 클래스에 의존합니다.

```html
<body class="theme-editorial accent-neutral">
<body class="theme-warm">
<body class="theme-bold accent-swap">
```

## 트러블슈팅

| 문제 | 원인 | 해결 |
|------|------|------|
| 한글 깨짐 | 폰트 미설치 | `fc-list :lang=ko` 확인, Noto CJK 설치 |
| 이미지 잘림 | height 강제 지정 | height 옵션 제거 (자동 계산) |
| 외부 리소스 에러 | 네트워크 차단 | CSS/폰트 로컬/인라인 처리 |
| 여백 과다 | body margin | `body { margin: 0; }` 확인 |
| 변형 어색 | 톤 감지 오류 | 사용자에게 테마/변형 직접 선택 제시 |
| 컬러 부조화 | 홈페이지 컬러 추출 오류 | 수동 HEX 입력으로 대체 |
